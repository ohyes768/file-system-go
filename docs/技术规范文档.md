# 音频文件服务器技术规范文档

## 1. 项目概述

### 1.1 项目名称
Go Audio File Server

### 1.2 项目目标
基于 Go 语言开发的轻量级音频文件上传服务,用于在阿里云 ECS 服务器上托管音频文件,提供文件上传、下载和静态文件服务功能。

### 1.3 技术选型理由
- **Go 语言**: 单文件部署、高性能、跨平台、低资源占用
- **Gorilla Mux**: 强大的 HTTP 路由器
- **YAML 配置**: 灵活的配置管理
- **双日志输出**: 控制台 + 文件日志

## 2. 需求分析

### 2.1 功能需求

#### 2.1.1 核心功能
1. **健康检查接口**
   - 路径: `GET /`
   - 响应: 服务状态、上传端点、音频目录信息

2. **文件上传接口**
   - 路径: `POST /upload`
   - 功能: 接收 multipart/form-data 格式的文件上传
   - 限制: 最大 100MB
   - 响应: 上传成功后的文件名、URL、大小

3. **静态文件服务**
   - 路径: `/audio/{filename}`
   - 功能: 提供已上传音频文件的 HTTP 访问

#### 2.1.2 非功能性需求
1. **性能**: 高并发文件上传和下载
2. **可靠性**: 日志记录、错误处理、服务自动重启
3. **可维护性**: 清晰的代码结构、配置文件管理
4. **可部署性**: 单文件部署、交叉编译支持

### 2.2 约束条件
- Go 版本: 1.21+
- 部署环境: Linux AMD64 (阿里云 ECS)
- 本地开发: Windows 环境
- 配置管理: YAML 配置文件
- 日志输出: 控制台 + 文件 (logs/ 目录)
- Go 代理: 国内环境使用 GOPROXY 加速依赖下载

### 3.3 接口设计

#### 3.3.1 健康检查接口
- **方法**: GET
- **路径**: `/`
- **请求**: 无
- **响应**:
  ```json
  {
    "service": "Audio File Server (Go)",
    "status": "running",
    "version": "1.0.0",
    "upload_endpoint": "/upload",
    "audio_dir": "/var/www/audio"
  }
  ```

#### 3.3.2 文件上传接口
- **方法**: POST
- **路径**: `/upload`
- **请求**: multipart/form-data
  - 字段名: `file`
  - 文件类型: audio/*
- **成功响应** (200):
  ```json
  {
    "success": true,
    "filename": "test_audio.wav",
    "url": "/audio/test_audio.wav",
    "size": 12345
  }
  ```
- **失败响应** (400/500):
  ```json
  {
    "success": false,
    "error": "错误描述"
  }
  ```

#### 3.3.3 静态文件访问
- **方法**: GET
- **路径**: `/audio/{filename}`
- **响应**: 文件二进制内容

#### 3.3.4 文件检查接口（新增 v1.1.0）
- **方法**: GET
- **路径**: `/api/check/{filename}`
- **功能**: 检查文件是否存在
- **请求参数**:
  - `filename`: 文件名（路径参数）
- **成功响应** (200):
  ```json
  {
    "exists": true,
    "filename": "7123456789012345678.mp4",
    "size": 102400000,
    "upload_time": "2026-02-26T12:00:00Z"
  }
  ```
- **文件不存在响应** (200):
  ```json
  {
    "exists": false,
    "filename": "7123456789012345678.mp4"
  }
  ```
- **用途**: 支持客户端增量处理，跳过已上传的文件

## 5. 技术实现

### 5.1 依赖包
```go
require (
    github.com/gorilla/mux v1.8.1          // HTTP 路由
    gopkg.in/yaml.v3 v3.0.1                // YAML 解析
)
```

### 5.2 关键技术点

#### 5.2.1 配置管理
- 使用 `gopkg.in/yaml.v3` 解析 YAML 配置
- 环境变量覆盖支持 (可选)
- 配置热加载 (不实现,保持简洁)

#### 5.2.2 日志系统
- 使用标准库 `log` 包
- 自定义 `io.MultiWriter` 实现双输出
- 日志文件按日期轮转

#### 5.2.3 文件上传
- 使用 `http.MaxBytesReader` 限制大小
- `r.ParseMultipartForm()` 解析表单
- `io.Copy` 高效复制文件
- 错误处理和资源清理 (defer Close)

#### 5.2.4 静态文件服务
- 使用 `http.FileServer`
- `http.StripPrefix` 去除路径前缀
- 支持范围请求 (断点续传)

#### 5.2.5 中间件
- 日志中间件: 记录请求方法、路径、来源、耗时
- 恢复中间件: 捕获 panic,防止服务崩溃
- CORS 中间件 (可选)

#### 5.2.6 Go 代理配置（国内环境）
- **问题**: 国内访问 `proxy.golang.org` 速度慢或超时
- **解决方案**: 配置国内 Go 代理镜像
- **推荐代理**:
  - 七牛云: `https://goproxy.cn`
  - 阿里云: `https://mirrors.aliyun.com/goproxy`
- **配置方式**:
  ```bash
  # 临时配置
  export GOPROXY=https://goproxy.cn,https://mirrors.aliyun.com/goproxy,direct
  export GO111MODULE=on

  # 永久配置（添加到 ~/.bashrc）
  echo 'export GOPROXY=https://goproxy.cn,direct' >> ~/.bashrc
  source ~/.bashrc
  ```
- **编译脚本自动化**: `scripts/build.sh` 和 `scripts/build-test.sh` 已内置 GOPROXY 配置

## 7. 部署和运维

### 7.1 开发环境要求
- Go 1.21+
- Windows 10/11 (本地开发)
- Git (版本控制)

### 7.2 生产环境要求
- Linux (CentOS/Ubuntu/Alibaba Cloud Linux)
- AMD64 架构
- 80/8000 端口开放
- 足够磁盘空间

### 7.3 systemd 服务配置
```ini
[Unit]
Description=Audio File Server (Go)
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/audio-server
ExecStart=/root/audio-server
Restart=always
RestartSec=5s
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

### 7.4 日志管理
- 日志位置: `logs/` 目录
- 日志轮转: 按日期自动创建新文件
- 日志清理: 手动或定时任务清理旧日志
- systemd 日志: `journalctl -u audio-file-server`
